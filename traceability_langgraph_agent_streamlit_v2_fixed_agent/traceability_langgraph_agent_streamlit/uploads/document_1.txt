User Requirements Specification for HeartPulse mobile heart rate monitor android and ios app. This file is intentionally messy for extractor testing. no ids no bullets no table content is copied like a bad pdf export.
The system shall support heart rate capture using the phone camera and also supported wearable sensor. The system shall allow manual heart rate entry when the sensor is not available or reading failed. The system shall alert the user when heart rate is above or below configured limits. The system shall show history of readings with date time source and trend information. The system shall synchronize readings between android and ios account when internet is available. The system shall protect personal health data using secure login and role based access. The system shall display privacy notice and ask consent before storing or sharing heart rate information. The system shall allow the user to export a heart rate report as pdf or csv for doctor review.
